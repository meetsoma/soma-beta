# v0.8.0 → v0.8.1

**What changed:**
- Warm content format unified: `## TL;DR` replaces `<!-- digest:start/end -->` for muscles
- Automations and MAPs unified: MAP is a type of automation, `map` accepted as install alias
- `discoverMaps()` now also scans `amps/automations/` root (installed automations visible as MAPs)
- `extractTldr()` shared across protocols, muscles, automations (muscles check TL;DR first, digest fallback)
- Hub validator updated: accepts `description`/`triggers`/`tags`, prefers `## TL;DR` over `<!-- digest -->`
- Settings: `keepalive` section (maxPings, autoExhale, autoExhaleMinTokens)
- Keepalive limited to 5 pings per idle period, resets on user message
- Auto-exhale: agent writes preload when keepalive expires (if context > 75k tokens)

**Actions:**
1. Add `keepalive: { maxPings: 5, autoExhale: true, autoExhaleMinTokens: 75000 }` to settings if missing
2. Convert muscle `<!-- digest:start -->...<!-- digest:end -->` blocks to `## TL;DR` sections
3. Code changes are automatic (ship with runtime update)
4. Set version to `0.8.1`

**Digest → TL;DR migration (Tier 1 — safe, idempotent):**
Recursively scans both `amps/muscles/` and `amps/automations/` (up to 2 levels deep,
skipping `_`-prefixed and `.`-prefixed directories — matches `discoverContent` behavior).

For each `.md` file found:
- If file has `<!-- digest:start -->` but no `## TL;DR`:
  - Extract content between digest markers
  - Strip leading `> ` from blockquote lines
  - Replace the digest block with `## TL;DR\n\n{content}`
- If file already has `## TL;DR`: skip (already migrated)
- If file has neither: skip (no warm content)

Covers muscles in subdirectories (e.g. `muscles/public/`, `muscles/dev/`)
and automation files with old digest markers.

Custom content is safe — the conversion preserves the text exactly,
just changes the container format.
