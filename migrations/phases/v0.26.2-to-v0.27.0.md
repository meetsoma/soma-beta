# v0.26.2 → v0.27.0

## Actions

This migration runs automatically at boot. No manual steps required.

## What changed

### breathe.auto: boolean → tri-state enum (Cycle 16)

The `breathe.auto` setting now accepts a tri-state string instead of a boolean:

| Old value | New value | Behavior |
|-----------|-----------|----------|
| `true` | `"global"` | Proactive rotation at fixed `triggerAt`/`rotateAt` percentages |
| `false` | `"off"` | Passive notifications only (`context.*` thresholds) |
| (missing) | `"model-aware"` | Per-model thresholds (new default for fresh installs) |

**Why**: Curtis hit Sonnet's `extra usage required for long context` 429 at 48% context in s01-8b3cb3. Both existing threshold systems (`breathe.triggerAt:50`, `context.notifyAt:50`) fire *after* the wall. A third, model-aware mode fires *before* the wall using empirically-validated per-model brackets.

### New: breathe.thresholds map

A `thresholds` map is added to all existing `breathe` configs if missing:

```jsonc
"thresholds": {
  "default":  { "warnRange": [50, 64], "exhaleRange": [65, 85] },
  "*sonnet*": { "warnRange": [28, 33], "exhaleRange": [34, 50] },
  "*opus*":   { "warnRange": [60, 74], "exhaleRange": [75, 90] }
}
```

Glob patterns (`*sonnet*`) match case-insensitively against `ctx.model.id`. First match wins; `"default"` is the fallback. The Sonnet thresholds fire before the ~40-48% empirical long-context wall on Curtis's account.

### Migration behavior

The `breathe-tri-state-v0.27.0` sentinel gates this migration (recorded in `settings.migrations[]`). It runs once on the first boot after upgrade:

- **Existing `breathe.auto: true`** → changed to `"global"`. Your `triggerAt`/`rotateAt` values are preserved and continue to drive behavior.
- **Existing `breathe.auto: false`** → changed to `"off"`. Passive notifications continue as before.
- **`breathe.auto` missing** → full model-aware defaults seeded.
- **`thresholds` missing from any `breathe` object** → default thresholds map injected.

### Settings impact

```jsonc
// Before (v0.26.2 and earlier)
"breathe": {
  "auto": false,
  "triggerAt": 50,
  "rotateAt": 70,
  "graceSeconds": 30
}

// After (v0.27.0 — existing install with auto: false)
"breathe": {
  "auto": "off",           // migrated from false
  "triggerAt": 50,
  "rotateAt": 70,
  "thresholds": { ... },   // injected
  "graceSeconds": 30
}
```

### New /auto-breathe command variants

The `/auto-breathe` command now accepts `off | global | model-aware | status`:

```
/auto-breathe                → status (shows current mode + resolved thresholds for current model)
/auto-breathe off            → disable (passive warnings only)
/auto-breathe global         → fixed thresholds (triggerAt/rotateAt)
/auto-breathe model-aware    → per-model thresholds
```

`/auto-breathe on` continues to work as an alias for `global` (backward compat).
