# v0.10.0 → v0.11.0

## What Changed
- **Identity: soul.md is primary** — new projects create `body/soul.md` instead of `SOMA.md`. Existing SOMA.md/identity.md still work as fallbacks.
- **Templates moved** — `body/_public/` deleted, templates now at `templates/default/`.
- **autoInject default: false** — new projects require `soma inhale` for preload loading. Existing settings preserved.
- **First-run minimal boot** — first session has minimal system prompt (no hot protocols/muscles).
- **Pi 0.65 migration** — `session_switch`/`session_fork` events removed, replaced by `session_start` with `event.reason`.
- **Protocol heat-defaults** — only breath-cycle and working-style start warm. All others cold.
- **11 bundled scripts** (was 6) — added soma-body, soma-refactor, soma-reflect, soma-plans, soma-session.
- **Extension allowlist** — configure in settings.json, `/soma doctor` reports unlisted.
- **Image payload guard** — strips old images when >15 accumulate, pauses keepalive on 400 errors.
- **Discovery** — `findSomaDir()` detects `body/` as a marker (soul.md-only projects).

**Actions:**
- Seed new scripts if missing (doctor handles this automatically)
- Update stale body templates (_first-breath.md has updated CLI syntax)
- No settings changes needed (existing autoInject preserved)
- Protocol heat-default changes only affect new `bootstrapProtocolState` runs (existing state.json is not reset)

**Compatibility:**
- SOMA.md continues to work — loadIdentity checks soul.md first, falls back to SOMA.md, then identity.md
- body/_public/ references in user code should be updated to templates/default/
- session_switch/session_fork handlers in custom extensions must migrate to session_start with event.reason
