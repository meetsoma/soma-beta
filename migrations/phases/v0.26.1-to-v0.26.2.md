# v0.26.1 → v0.26.2

## Actions

This migration runs automatically. No manual steps required.

## What changed

### New setting: `extraUsageRecovery: "auto"` (default)

Narrowly-scoped recovery for the boot-turn-after-`/inhale` variant of Anthropic's `extra usage` 400 — the variant the SX-727/SX-734 burns deliberately did NOT touch. Cycle 13 successor (s01-c62a62).

**What the user sees:**

When `/inhale` rotates the session and the first API call returns Anthropic's `"You're out of extra usage. Add more at claude.ai/settings/usage and keep going."`:

- **`"auto"` (default)** — single notify + Soma auto-injects `.` after a 1s debounce. Pi's next turn cache-reads + 1-char content; subsequent turns proceed normally.
- **`"notify"`** — same notify, no auto-action. User sends any message to continue.
- **`"off"`** — silent. Pi's raw error display passes through unmodified.

**Notify text (default `"auto"`):**

> ⓘ Anthropic returned an extra-usage notice. Long-context requests may be billed as extra usage. Auto-continuing. (https://claude.ai/settings/usage)

### Hard exclusions (cycle 13 plan baked in)

The recovery branch fires ONLY when ALL three conditions hold:

1. `err.includes("extra usage")` — literal Anthropic phrase
2. `state.turnCount <= 2` — only first call after rotation; silent rest of session
3. `state.consecutiveKeepalives === 0` — not a keepalive-fired error

Auto-injection is `"."` (1 character). **No full-context resend.** Pi's next turn cache-reads + tiny content, so there is no SX-709-style 2× billing amplifier — that's what killed the original auto-retry.

The recovery is a sibling of the existing `rate_limit` / `429` / `invalid_request_error` / `image dimensions exceed` branches in `extensions/soma-statusline.ts:892-960` — same handler, same fence shape, new specific case.

## Settings impact

`addIfMissing("extraUsageRecovery", "auto")` runs once at boot for installs that don't have the key yet. Existing installs keep their value if set. The decorative `retry: { enabled: false }` key is **not** removed (left alone for backward-compat with installs that have it; never consumed in code).

## Migration record

No migration ID needed — this is a pure additive default via `addIfMissing`. The `migrations[]` sentinel array doesn't grow for this one (the array is for one-time semantic transforms, not default-seeding).

## Why this is NOT a retry

Pi's `_isRetryableError` (`agent-session.js:1921`) deliberately excludes `extra usage` from its retry regex — that decision is correct for a billing-tier rejection. The auto-`.` is recovery (advance the conversation past the error so cache-state can settle for smaller subsequent turns), not retry (which would re-fire the same heavy boot context and either burn extra credit or fail again with the same error).
