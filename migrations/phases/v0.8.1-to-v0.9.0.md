# Migration: v0.8.1 → v0.9.0

## What Changed

### Settings Additions
- `inherit` section — controls AMPS inheritance chain (identity, protocols, muscles, tools)
- `keepalive` section — controls cache ping behavior (maxPings, autoExhale, autoExhaleMinTokens)
- `heat.autoDetectBump` — default 1, controls how much auto-detected usage bumps heat

### CLI Script Routing
Scripts are now accessible via `soma <name>` instead of `soma-<name>.sh`:
- `soma code map <file>` (was `soma-code.sh map <file>`)
- `soma verify` (was `soma-verify.sh`)
- `soma refactor scan` (was `soma-refactor.sh scan`)
- `soma focus <keyword>` (was `soma-focus.sh <keyword>`)

The `.sh` scripts still work directly, but the CLI routing is the preferred interface.

### AMPS Consolidation
- Community repo is canonical for protocols and muscles
- Agent repo is canonical for scripts
- Duplicate protocols/muscles deleted (~37K lines)
- `_` prefix convention: `_dev/`, `_archive/`, `_private/`, `_local/` — invisible to runtime

### New Scripts
- `soma-verify.sh` — comprehensive project health checks
- `soma-refactor.sh` — dependency scanning, route audits
- `soma-browser.sh` — headless browser automation

## Migration Steps

**Actions:**

### Automatic (soma doctor --fix)
1. Add missing `inherit` section to settings.json
2. Add missing `keepalive` section to settings.json  
3. Copy any new bundled scripts to `.soma/amps/scripts/`
4. Update bundled protocols to latest versions (preserving heat)
5. Bump version field to 0.9.0

### Manual (optional)
- Delete duplicate protocols if you see `_public/` copies alongside originals
- Update any custom scripts/MAPs that reference `soma-<name>.sh` to `soma <name>`
- Run `soma verify` to check project health
