# v0.28.0 → v0.28.1

## Actions

### Template: `_memory.md` — new "Check for Exhale Note" section

The bundled `_memory.md` template has a new section at the top that tells the agent how to handle the `### Note` block from `/exhale <note>`. The doctor will detect this drift on next boot if you have a custom `_memory.md` and flag it. No auto-overwrite — your customizations are preserved.

### Settings: none

No settings changes in this release.

## What changed

### `/exhale <note>` now works (89efd1b6, s01-14580a)

Text after `/exhale` (e.g. `/exhale skip the deploy step`) is injected as a `### Note` block in the EXHALE follow-up. The agent uses it to:
- **Scope the wrap** — "quick" / "minimal" skips body audit + MLR steps
- **Pass directives forward** — "tell the next agent X" puts that in Start Here

Previously, text after `/exhale` was silently dropped. The `--note` prefix and surrounding quotes are stripped automatically. `/exhale` alone (no text) works the same as always.

### `soma inhale --model <model>` preload loading fixed (s01-14580a)

The CLI's nameArg extraction was stealing model values as preload names. Now fixed — model values from `--model` are excluded from the preload name search.

### Full preload format at all context thresholds (1ae67a38)

High-context preloads now use the full structured template instead of a minimal format that lost context.

### Release: Step 6 main-sync hard gate (3c51de02)

Release pipeline Step 6 now exits 1 on push failure — no more "published but stale runtime" like v0.28.0.

### Template drift

The doctor's `compareTemplates` will flag `_memory.md` as drifted if you have a custom copy. This is expected and not an error — the new section is additive.
