# v0.32.1 → v0.32.2

## Actions

### Settings: `guard.trustedModels` (new, default `[]`) — SX-787

Per-model guard allowlist. When the active model id matches a glob in this list
(`["*sonnet*", "*opus*"]`), `coreFiles` + `bashCommands` resolve to `"allow"` for that
turn — capable models skip the prompts while weaker models and new users keep full
protection. Empty (default) = no model trusted → no behavior change. Settable per-project
or global; child wins.

### Settings: `guard.bashNotify` (new in schema, default `"notify"`) — SX-788

The non-blocking bash notice toggle (`"notify"` | `"off"`). It was already *consumed* by
`soma-guard.ts` but was missing from the settings interface + DEFAULTS (untyped `as any`
access). Now typed + defaulted; no behavior change for existing users (read-time default
was already `"notify"`).

### Guard behavior: always-on irreversibility tier — SX-787 Option A

`.soma`/`.git` destruction, `git init`, runtime-destroy, and expensive ops now confirm
**regardless** of `bashCommands`/`trustedModels`. No settings change — a behavior fix.
No migration action required; documented here for completeness.
