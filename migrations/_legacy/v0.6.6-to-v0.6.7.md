# Migration: v0.6.6 → v0.6.7

## What Changed

### Settings
- `systemPrompt.maxTokens` default raised from 4000 to 10000.
  The old default triggered warnings on fresh installs (~4056 tokens with
  just bundled protocols). Anthropic's system prompt is ~25,000 tokens;
  Soma at ~5000 is already efficient. 10,000 gives room to grow.

### Init UX
- `detectProjectContext()` now distinguishes global `~/.soma/` from real parent workspaces.
- Init prompt messaging differs: global fallback vs parent choice.
- Declining init with only global → session uses global (was: null/broken).

### Body Templates
- `_mind.md`: Added section headers (`## Where I Am`, `## Voice`, `## How to Behave`, etc.)
  so the compiled system prompt has visible structure.
- `DNA.md`: Slimmed from ~400 lines to ~100 lines. Essential reference only.
- `soul.md`: Cleaner starter with guidance comments.

### Protocols
- 14/17 bundled protocol TL;DRs rewritten from spec voice to coaching voice.

## Migration Logic

### Safe to auto-fix (script handles):
1. **`systemPrompt.maxTokens`** — numeric field, always safe to update.
2. **`_mind.md`** — if content matches v0.6.4/v0.6.5/v0.6.6 bundled default → replace.
   Template files are structural, not personal. User rarely edits these.
3. **`DNA.md`** — if content matches old bundled default → replace.
   Reference doc, not identity.
4. **Protocols** — if file content matches old bundled version → replace with new.
   If user has modified → skip, log.
5. **Version bump** — always last.

### Needs agent judgement:
1. **`soul.md`** — if user has customized, don't touch. Only replace if it's
   still the v0.6.4 boilerplate word-for-word.
2. **`voice.md`** — same: only replace if unmodified default.

### Detection: "is this file modified?"
Compare against known old defaults. The script ships fingerprints (md5 of
stripped content) for each version's bundled templates. If the file matches
any known default → safe to replace. If not → user modified it, skip.

## Fingerprints (old defaults to detect)

Script will compute `md5 -q` of the frontmatter-stripped content and compare
against known hashes from v0.6.4 and v0.6.5 bundled files. This is more
reliable than string matching and handles minor whitespace differences.
